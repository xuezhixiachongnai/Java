# MQ

### MQ 的应用场景

#### 异步处理

传统处理任务的方式有 **串行方式**、**并行方式**，这两种方式都需要等待所有模块将任务执行完毕才算完成任务

```sh
模块1 -----> 模块2 -----> 模块3
		  ----> 模块2
	     |
模块1 ---------> 模块3
```

如果使用了 mq，我们可以将任务发送到 mq 直接返回，因为已经将核心任务完成，无需等待非核心任务

```sh
模块1 -----> mq -------> 模块2
                  |
                   ---> 模块 3
```
